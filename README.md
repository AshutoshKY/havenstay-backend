# Airbnb Clone Backend

This repository contains a backend system for a hotel and room booking service (Airbnb clone) built in **GoLang**. It utilizes a strict **Domain-Driven Design (DDD)** architecture exposing both **REST** and **gRPC** interfaces, backed by **MySQL**.

## Architecture & DDD Layers

The application is split into specialized bounded contexts (`domains`). Each domain strictly adheres to the translation between external boundaries (gRPC/HTTP), business logic (`Entities`), and storage (`Models`).

```mermaid
graph TD
    Client((Client)) <-->|HTTP JSON| REST[Gin REST Server]
    Client <-->|HTTP/2 Protocol Buffers| GRPC[gRPC Server]
    
    REST -->|v1.*Request (Protos)| Core
    GRPC -->|v1.*Request (Protos)| Core
    
    subgraph Domain: Property
        Core[Core Service] -->|Entity (Business logic)| Repo[MySQL Repository]
        Repo -->|GORM Model| DB[(MySQL)]
        Repo -->|Entity| Core
        Core -->|v1.*Response (Protos)| REST
    end
```

### The Request Flow
1. **Controller Layer (Server)**: A REST handler or gRPC server receives the external request payload (struct auto-generated by Protobuf `*.pb.go`).
2. **Core Layer (Service)**: The payload is handed to the domain Service interface.
3. **DTO Mapping**: The service maps the `proto` request into an internal abstract business `Entity`.
4. **Repository Layer**: The core service requests data persistence/retrieval from the Repo interface using the `Entity`.
5. **Storage mapping**: The repository converts the domain `Entity` into a GORM `Model`, persists it, and returns the converted `Entity` back to the Core.

## Tech Stack
* **Language**: Go
* **Database**: MySQL 8.0, interacting through GORM.
* **Network Frameworks**: `gin-gonic` (HTTP), `grpc-go` (gRPC).
* **Code Generation**: Google Protobuf (`protoc`).
* **API Documentation**: `swaggo/swag` (Swagger generation).

## Running the Project

### 1. Requirements
- Docker & docker-compose
- Go 1.21+
- `make`

### 2. Boot Database & Seed Data
A script is provided to automatically create tables and populate them with dummy users, properties, and bookings.

```bash
# 1. Start the MySQL container in the background
docker-compose up -d

# 2. Run the migration and seed script
go run cmd/seed/main.go
```

### 3. Build & Run the Backend
```bash
# Optional: compile the protobufs if you edit api/proto/v1/
make proto

# Boot the API server (Runs HTTP on :8080 and gRPC on :50051)
go run cmd/server/main.go
```

## API Documentation

Automated OpenAPI docs are provided via Swaggo! 
When the server is booted, the interactive UI is accessible via the browser at:
**[http://localhost:8080/swagger/index.html](http://localhost:8080/swagger/index.html)**

### Postman Support
The root folder contains a `postman_collection.json`. You can immediately import this exact file into Postman to have all requests configured for local testing.

```text
airbnb-test/
├── api/
│   └── proto/          # Protobuf definitions generating network interfaces
├── cmd/

├── internal/

│   │   └── host/
│   │       ├── model.go    (GORM Mapping)
│   │       ├── entity.go   (Abstract Structure)
│   │       ├── dto.go      (Proto <-> Entity <-> Model translation)
│   │       ├── repo.go     (Persistence Interface)
│   │       ├── core.go     (Business Interface)
│   │       └── server.go   (gRPC/HTTP Routers)
│   └── platform/       # Global/Generic logic mapping
│       ├── database/   # MySQL Init
│       ├── migration/  # GORM auto-migration
│       └── server/     # Top level router attachments (Gin, gRPC)
└── docker-compose.yml  # Local stack containerization
```
